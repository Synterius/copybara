# Copybara — Cyber Review

## 1. Загальний опис

Copybara — локальна десктопна програма для організації текстових команд та інструкцій у `.txt` файлах.

Програма призначена для швидкого пошуку, редагування, структурування та копіювання заздалегідь підготовлених текстових блоків. Copybara не виконує збережені команди, а лише копіює їх у буфер обміну користувача.

Технологічний стек:

* Tauri 2
* React
* TypeScript
* Material UI
* Rust backend shell через Tauri

## 2. Основна функціональність

Copybara дозволяє користувачу:

* вибрати робочу папку через системний dialog;
* читати дерево папок і `.txt` файлів;
* відкривати `.txt` файли;
* створювати нові `.txt` файли;
* створювати папки;
* перейменовувати файли та папки;
* видаляти файли та папки після підтвердження;
* редагувати текстові блоки всередині `.txt` файлу;
* копіювати текст інструкції в буфер обміну;
* шукати інструкції;
* виконувати локальну заміну тексту перед копіюванням;
* перевіряти наявність оновлень;
* встановлювати підписані оновлення через штатний Tauri updater.

## 3. Що програма не робить

Copybara не виконує збережені команди.

Програма не містить логіки:

* запуску shell-команд;
* запуску зовнішніх процесів;
* виконання PowerShell, cmd, bash або sh;
* відкриття TCP/UDP портів;
* прослуховування мережевих підключень;
* WebSocket-зʼєднань;
* HTTP API-запитів з власного коду;
* збору паролів;
* збору токенів;
* збору cookies;
* доступу до keychain/credential storage;
* прихованої телеметрії;
* відправки вмісту `.txt` файлів на зовнішні сервери.

## 4. Файлова система

Copybara працює з файловою системою тільки в межах робочої папки, яку користувач самостійно вибирає через системний dialog.

Основні операції:

* читання директорії;
* читання `.txt` файлів;
* запис `.txt` файлів;
* створення папок;
* перейменування файлів/папок;
* видалення файлів/папок після підтвердження користувачем.

Назви файлів і папок проходять валідацію.

Заборонені:

* порожні назви;
* `.` і `..`;
* назви з `/` або `\`;
* назви з `..`;
* символи `< > : " | ? *`.

Для файлів додатково контролюється `.txt` розширення.

## 5. Tauri permissions

Permissions були звужені до мінімально необхідних.

Використовуються такі категорії дозволів:

### Core/window

Потрібні для базової роботи вікна:

* показ вікна;
* встановлення title;
* мінімізація;
* режим always-on-top.

### File system

Потрібні для роботи з обраною користувачем робочою папкою:

* читання директорії;
* читання текстових файлів;
* запис текстових файлів;
* створення папок;
* перейменування;
* видалення.

### Dialog

Потрібний для вибору робочої папки.

### Opener

Використовується тільки для функції “Показати у провіднику”.

### Updater

Звужено до:

* перевірки оновлень;
* завантаження та встановлення оновлень.

### Process

Звужено до перезапуску програми після оновлення.

### Persisted scope

`tauri-plugin-persisted-scope` використовується для збереження дозволу на доступ до робочої папки, яку користувач сам вибрав через системний dialog.

Без цього плагіна Copybara не може після перезапуску повторно відкрити раніше вибрану папку.

## 6. Мережева активність

У власному коді Copybara не знайдено:

* `fetch`;
* `axios`;
* `XMLHttpRequest`;
* `WebSocket`;
* `TcpListener`;
* `TcpStream`;
* `UdpSocket`;
* `reqwest`.

Єдиний production URL у конфігурації — endpoint Tauri updater:

`https://raw.githubusercontent.com/Synterius/copybara-releases/main/latest.json`

Цей endpoint використовується для перевірки наявності оновлень.

У dev-режимі використовується локальний Vite dev server:

`http://localhost:1420`

Це не використовується у production-збірці.

## 7. Оновлення

Copybara використовує штатний Tauri updater.

Оновлення перевіряються через updater manifest.

У конфігурації задано публічний ключ перевірки підпису оновлень. Це означає, що updater не повинен встановлювати непідписані або некоректно підписані пакети.

Мережева активність updater обмежена перевіркою manifest-файлу та завантаженням підписаного оновлення.

## 8. Content Security Policy

Увімкнено CSP замість `csp: null`.

Поточна політика:

```text
default-src 'self';
img-src 'self' asset: http://asset.localhost;
style-src 'self' 'unsafe-inline';
script-src 'self'
```

`unsafe-inline` залишено для стилів, оскільки UI використовує Material UI / runtime-generated styles.

Для script використовується тільки `'self'`.

## 10. Перевірка залежностей

### npm audit

Команда:

```powershell
npm audit --omit=dev --audit-level=high
```

Результат:

```text
found 0 vulnerabilities
```

### cargo audit

Команда:

```powershell
cargo audit
```

Результат: знайдено warnings по транзитивних залежностях.

Характер warnings:

* unmaintained GTK3 bindings;
* unmaintained транзитивні crates;
* один warning по `bincode`, який приходить через `tauri-plugin-persisted-scope`;
* `glib` warning через Linux GUI стек.

Більшість warnings приходять через стандартний GUI/runtime ланцюжок Tauri/Wry/GTK/WebKit, а не через власний код Copybara.

Copybara не використовує ці crates напряму.

Ризик знижено за рахунок:

* локального характеру програми;
* відсутності виконання команд;
* відсутності власної мережевої логіки;
* відсутності обробки credential-даних;
* звужених Tauri permissions;
* CSP;
* підписаного updater;
* валідації назв файлів і папок.

## 11. Висновок

Copybara є локальним desktop-інструментом для роботи з `.txt` файлами та копіювання текстових інструкцій.

Програма має прозору поведінку:

* працює з папкою, яку вибрав користувач;
* не виконує збережені команди;
* не має прихованої мережевої логіки;
* не відкриває порти;
* не збирає паролі, токени або cookies;
* не відправляє вміст файлів на сервер;
* використовує підписаний механізм оновлення;
* має звужені Tauri permissions;
* має базову CSP;
* проходить npm audit без high production vulnerabilities.

Відомі cargo audit warnings повʼязані переважно з транзитивними залежностями Tauri/Wry/GTK/WebKit і мають бути розглянуті як обмеження поточного Tauri/Linux GUI runtime, а не як власна небезпечна логіка Copybara.
